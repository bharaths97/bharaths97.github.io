Kubernetes — Crash Course + Your Story

What It Is

Container orchestration system. You define what you want running; K8s makes it happen across a cluster of machines and keeps it alive. Think of it as the operating system for distributed infrastructure.

Strike runs their backend microservices (payments, Lightning nodes, lending logic, APIs) as containers orchestrated by K8s on GCP's managed service — GKE (Google Kubernetes Engine).

Key Components — Know These

Component What it is / Why it matters for security
Pod Smallest deployable unit. One or more containers. Compromise a pod, you're inside the cluster.
Node A machine running pods. Compromising a node means compromising all pods on it.
Control Plane The cluster brain — API server, scheduler, etcd (cluster state DB). If etcd is unencrypted, secrets are exposed.
kubectl CLI to interact with the cluster. Misconfigured RBAC + kubectl access = significant blast radius.
RBAC Role-Based Access Control. Most critical security control. Over-permissioned service accounts are extremely common.
Namespace Logical isolation within a cluster. Often assumed to be stronger than it is — it's not a security boundary by default.
Service Account Identity assigned to pods for API access. Abused for lateral movement if over-permissioned.
Network Policy Controls pod-to-pod traffic. Missing by default — pods can reach anything without it.
Secrets K8s-native credential store. Often base64-encoded, not encrypted. Stored in etcd unencrypted if not configured.
Ingress Controller Routes external traffic in. Where WAF rules (Cloudflare) sit upstream.

Attack Surfaces

RBAC misconfiguration

Service accounts with cluster-admin, wildcard verbs, or cross-namespace permissions. Most common K8s security issue.

Metadata API abuse

Pods hitting 169.254.169.254 (cloud metadata endpoint) can steal node credentials and pivot to cloud IAM. Classic SSRF from container to AWS/GCP keys.

Privileged containers

Containers running as root or with --privileged flag. Container escape to the host node becomes trivial.

Secrets in etcd

If encryption-at-rest isn't configured, all K8s Secrets are base64 in etcd. Direct read = full credential dump.

Missing network policies

Default: every pod can reach every other pod. Compromise one service, pivot laterally to payment logic or DB pods.

Supply chain

Unverified images from public registries. Malicious or outdated base images running in production.

What Cloudflare WAF Rules Do — Basics

  • Managed rules: Cloudflare's pre-built ruleset covering OWASP Top 10, known CVEs, common payloads.
  • Custom rules: You write expressions to match specific request patterns — IP, path, headers, body content, rate.
  • Rate limiting: Throttle requests from IPs hitting endpoints too fast. Critical for payment APIs.
  • Bot management: Score-based filtering. Relevant for Strike's buy/sell endpoints being scraped or abused.

Where WAF Rules Matter for Strike

  • Lightning payment endpoints — rate limiting to prevent abuse
  • Authentication endpoints — block credential stuffing patterns
  • API endpoints — filter malformed or oversized payloads
  • Lending endpoints — high-value targets, stricter geo and rate rules

Strike's Attack Surface — Think Like the CISO

Strike handles real money. Bitcoin custody, Lightning payments, BTC-backed loans up to $2M. The consequence of a security failure is not a data breach — it's customer funds. The CISO thinks in these terms.

Authentication endpoints

Account takeover = access to BTC balance. Credential stuffing, session hijacking, MFA bypass — high priority targets.

Lightning Network nodes

Channel management, routing, payment state. Malformed payment routing could drain channels. Novel attack surface.

Lending platform

Bitcoin collateral management, liquidation logic. Business logic flaws here can be massively impactful. Launched May 2025.

KYC / identity layer

NYDFS regulated. Identity fraud, account takeover, synthetic identities. Compliance failure = license risk.

API layer

Buy/sell/send/borrow endpoints. Rate limiting, authorization between accounts, IDOR risks on transaction history.

Third-party capital providers

BTC collateral transferred to lending partners. Supply chain / trust boundary risk in the custody handoff.
